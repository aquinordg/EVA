# Summary

EVA (EEG data Validation and preprocessing Assistant) is an open-source
Python library that takes raw EEG recordings from any supported format
and produces preprocessed, labelled epoch arrays ready for downstream
statistical analysis or machine learning. The public API consists of
three functions: `convert()` transforms any supported format
(BrainVision, EDF, BDF, EEGLAB) into MNE `.fif`; `preprocess()` applies
a configurable filter chain to the continuous signal, segments it into
epochs aligned to event annotations, and saves the result as a
compressed HDF5 file; and `sync()` appends behavioural and
physiological co-variables to an existing `.h5` file, keeping all data
for one participant in a single container ready for multimodal analysis.
An optional grid-search optimiser (`find_best_params()`) identifies the
filter configuration that maximises a composite signal quality score
before full processing.

# Statement of need

Preprocessing raw EEG data into analysis-ready epochs involves a
repeated sequence of decisions — frequency cutoffs, reference scheme,
artefact thresholds — that are often implemented ad hoc in
laboratory-specific scripts. This fragmentation hinders reproducibility:
two researchers applying nominally identical preprocessing steps to the
same recording may obtain different results due to implementation
details that are rarely reported in full [@Luck2014].

Established toolboxes such as MNE-Python [@Gramfort2013] and EEGLAB
[@Delorme2004] provide comprehensive environments for EEG analysis but
require substantial configuration effort and domain expertise to assemble
a reproducible preprocessing workflow. EVA addresses this gap by wrapping
MNE's I/O and epoch extraction with a fixed, scientifically motivated
filter chain and a minimal three-function API. The library is designed
for researchers who need a reproducible, portable preprocessing step —
particularly in multimodal paradigms where EEG is acquired alongside
other modalities (e.g. eye tracking, behavioural responses) and must be
batch-processed against event markers generated by a separate stimulus
delivery system.

EVA was developed to support the VECA-EEG platform [@VECAEEG], a Unity 6
virtual reality application for cognitive assessment that broadcasts
Lab Streaming Layer (LSL) markers synchronised to eye-tracking trials.
Those markers are recorded by BrainVision Recorder and embedded in
`.vmrk` files, where they serve directly as EVA's epoch class labels —
no remapping required.

# Description

## Three-function workflow

EVA exposes three top-level functions that map directly to the stages of
a typical EEG analysis pipeline.

**`convert(path)`** reads an EEG recording in any supported format
(BrainVision `.vhdr`, EDF, BDF, EEGLAB `.set`) and writes an MNE `.fif`
file. When `report=True`, it additionally runs bad-channel detection on
the raw signal and generates an HTML quality report for inspection before
any preprocessing is applied. No channels are removed or modified by
`convert()` — all decisions are left to the researcher.

**`preprocess(source)`** accepts either a file path or an `mne.Raw` object
and applies the five-step filter chain described below to the continuous
signal, segments it into epochs using the recording's event annotations,
and saves the result as a compressed HDF5 file with the following
structure:

```
subject.h5
  /eeg/data          (n_epochs, n_channels, n_times)  float32
  /eeg/labels        (n_epochs,)                       int32
  /eeg/ch_names      (n_channels,)                     str
  /eeg/label_names   (n_classes,)                      str
  /eeg/label_codes   (n_classes,)                      int32
  /metadata/         sfreq, tmin, tmax
```

**`sync(path)`** appends co-variable arrays to an existing `.h5` file
produced by `preprocess()`. Behavioural signals (e.g. reaction time,
accuracy) are written to `/behavioral/<key>`; physiological signals
(e.g. ECG, EMG) are written to `/physio/<key>` with an optional
`sfreq` attribute. All arrays must share the same first dimension as
the stored EEG epochs. An `overwrite` flag controls whether existing
datasets can be replaced.

## Filter chain

`preprocess()` applies five sequential steps to the continuous raw signal:

1. **DC detrend** (`DCDetrend`) — subtracts the temporal mean per
   channel, stabilising IIR filter initialisation.
2. **Zero-phase Butterworth bandpass** (`ButterworthFilter`) — default
   1–40 Hz, order 4; `scipy.signal.sosfiltfilt` eliminates group-delay
   distortion [@Virtanen2020].
3. **IIR notch** (`NotchFilter`) — default 60 Hz (configurable for
   50 Hz mains), Q = 30.
4. **Common Average Reference** (`AverageReference`) — subtracts the
   instantaneous spatial mean, attenuating artefacts common to all
   electrodes.
5. **Soft clipper** (`SoftClipper`) — hyperbolic-tangent amplitude
   limiter; values within the threshold (default 100 µV) pass through
   unchanged, while excess amplitude is continuously compressed rather
   than zeroed, avoiding spectral discontinuities.

All parameters are keyword arguments to `preprocess()` and can be
overridden without modifying source code.

## Quality assessment

After filtering, EVA computes per-channel diagnostics using the
`QualityConfig` dataclass and `evaluate_all_channels()`. Channels are
evaluated against three criteria on the raw signal: flatness (std below
100 nV), peak amplitude (above 150 µV), and spectral outlier score
(Log-Spectra Deviation above 2.0). Channels triggering two or more flags
are classified as *bad*; one flag as *warning*; none as *good*. When
`convert()` is called with `report=True`, these criteria are applied
before any filtering so that bad channels can be excluded via
`channel_picks` in the subsequent `preprocess()` call.

At the recording level, EVA computes the PaLOSi index [@Hu2025] — a
measure of cross-spectral homogeneity that detects over-preprocessing or
dominant artefacts. Time-domain characterisation is provided via Hjorth
activity, mobility, and complexity [@Hjorth1970].

## Pipeline optimiser

`find_best_params(data, sfreq)` performs an exhaustive grid search over
candidate values for bandpass cutoffs, filter order, notch frequency,
artefact threshold, and optional processing steps. Each configuration
is scored by a composite metric:

$$\text{score} = \alpha \cdot \overline{\text{SNR}}_{\text{dB}} - (1 - \alpha) \cdot |\text{PaLOSi} - 0.45|$$

where $\alpha$ (default 0.5) balances reconstruction fidelity (SNR)
against spectral quality (PaLOSi). The penalty term drives the pipeline
toward the centre of the ideal PaLOSi range [0.3, 0.6] identified by
Hu et al. [@Hu2025] rather than blindly minimising the index. The
returned parameter dictionary can be unpacked directly into
`preprocess()`.

## Output format

Each recording produces a single `.h5` file (compressed with gzip,
level 4) compatible with h5py, NumPy [@Harris2020], and any HDF5-aware
data loader. The `sync()` function extends this file in-place so that
all data for one participant — EEG epochs, behavioural responses, and
auxiliary physiological signals — travel as a single self-contained
artefact through the analysis pipeline.
